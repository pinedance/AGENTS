## 작업 원칙

다음 작업 순서를 따를 것:
- 문제 원인 파악 
- (사용자 confirm 후) 문제 해결 계획 수립
- (사용자 confirm 후) 코드 수정 진행

코드 스타일:
* "Fail Fast" 적용: 지나친 예외 처리, 오류 회피 방지
* 코드 유지보수 재활용성 고려
* package를 사용할 수 있다면 직접 구현 대신 package 사용

코드 생성 및 수정:
* 항상 기억에 의존하지 말고 코드를 새로 읽고 작업 할 것 (사용자가 수정한 부분을 되돌리지 않도록)
* 기존 코드 및 커맨트 존중. 목적이 아닌 부분을 임의로 수정하지 않음 (사용자가 알지 못하는 문제 방지)
  * 코드를 커맨트 처리한 부분 남겨두기 ( 기억을 위해 의도된 것 )
* 사본을 만들어 사본에 수정된 코드 생성 (사용자가 전후를 비교할 수 있도록)

약속 기호:
* 코드 중에 "Logic:"으로 시작하는 주석은 해당 로직을 구현하는 코드를 생성해서 넣어 달라는 의미
